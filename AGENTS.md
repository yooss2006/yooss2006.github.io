# AGENTS.md

## 프로젝트

`rainy-notes`는 Astro 기반 정적 개인 블로그입니다.

- 블로그 이름: `비가 내리네요.`
- 목적: 개발 글과 일상 글을 오래 남기는 개인 기록장
- 배포 대상: `https://username.github.io` 형태의 GitHub Pages 사용자 사이트
- 패키지 매니저: `pnpm`

## 제품 방향

제품 기준 문서는 `docs/PRD.md`입니다.

핵심 원칙:

> 비는 배경이고, 글이 중심이다.

글을 쓰고 읽는 경험이 비 애니메이션, 분위기, 빗소리보다 항상 우선입니다.

## 구현 범위

현재 구현은 최소 Astro 프로젝트 설정에서 시작합니다.

명시적으로 요청받기 전까지 다음은 만들지 않습니다.

- 서버 런타임
- 데이터베이스
- 로그인
- 관리자 페이지
- CMS
- 댓글, 좋아요, 팔로우 같은 커뮤니티 기능
- 광고, 후원, 구독 같은 수익화 기능
- GitHub 원격 저장소 설정
- 실제 배포 설정
- 커스텀 도메인 설정

## GitHub Pages 기준

이 프로젝트는 `username.github.io` 저장소에 배포하는 것을 기본 전제로 합니다.

따라서 Astro 설정에는 `base`를 설정하지 않습니다.

프로젝트 저장소 배포 예시는 현재 범위가 아니므로, `base` 관련 설정이나 예시는 추가하지 않습니다.

## 명령어

패키지 매니저는 `pnpm`을 사용합니다.

```sh
pnpm install
pnpm dev
pnpm build
pnpm preview
```

로컬 에이전트 실행 중 사용자 홈 디렉터리에 Astro 텔레메트리 설정을 쓰지 않도록, `package.json` 스크립트에서는 Astro 텔레메트리를 비활성화합니다.

## 콘텐츠 원칙

글은 Markdown 또는 MDX 콘텐츠 파일 중심으로 관리합니다.

일반적인 새 글 작성은 코드 수정 없이 콘텐츠 파일 1개 작성으로 가능해야 합니다.

글 메타데이터는 `docs/PRD.md`의 `Post` 모델을 따릅니다.

- `title`
- `description`
- `publishedAt`
- `updatedAt`
- `category`
- `tags`
- `thumbnail`
- `draft`

## UX 원칙

- 미니멀한 텍스트 우선 디자인을 유지합니다.
- 넓은 여백과 안정적인 본문 가독성을 우선합니다.
- 비 애니메이션은 은은해야 하며 글보다 앞서면 안 됩니다.
- `prefers-reduced-motion` 설정을 존중합니다.
- 빗소리는 기본 꺼짐 상태여야 합니다.
- 오디오는 사용자가 명시적으로 켠 뒤에만 재생합니다.
- 소리 토글은 키보드로 접근 가능해야 합니다.
- 모바일에서는 장식보다 본문 폭, 행간, 여백을 우선합니다.

## 개발 규칙

의미 있는 구현 작업 전에는 다음을 지킵니다.

1. `docs/PRD.md`를 먼저 읽습니다.
2. 바로 개발하지 말고, 먼저 어떤 방식으로 만들지 사용자와 이야기합니다.
3. 제안에는 범위, 구조, 대안이나 트레이드오프를 짧게 포함합니다.
4. 사용자가 접근 방식에 동의하거나 명시적으로 구현을 요청한 뒤에만 파일을 수정합니다.
5. 작업이 설정 범위를 넘어갈 수 있으면 사용자에게 범위를 확인합니다.
6. 사용자가 dry-run을 선호하거나 요청한 경우, 파일 수정 전에 dry-run을 먼저 제시합니다.
7. 명시 요청이 없으면 Phase 1 범위 안에서 작게 변경합니다.
8. 구현 세부보다 글 작성 흐름이 가벼운지를 우선 판단합니다.

## 구현 우선순위

스타일 작업은 가장 마지막에 합니다.

초기 구현은 못생겨도 괜찮으므로, 글을 작성하고 읽는 흐름을 먼저 완성합니다.

우선순위:

1. 콘텐츠 컬렉션 안정화
2. 글 목록 페이지
3. 글 상세 페이지
4. 홈 최근 글 연결
5. 소개 페이지
6. 비 애니메이션
7. 빗소리 토글과 페이지 이동 중 지속 재생
8. 전체 스타일 다듬기

## 현재 상태

현재 저장소에는 최소 Astro 프로젝트 설정과 글 콘텐츠 컬렉션 기본 구조가 있습니다.

구현됨:

- Astro 설정
- `pnpm` 패키지 설정
- TypeScript 설정
- 기본 홈 페이지
- `.nojekyll`
- GitHub Pages 사용자 사이트 기준으로 `base`를 설정하지 않는 구성
- 글 콘텐츠 컬렉션
- 개발용 더미 글
- 글 상세 페이지
- 글 목록 페이지

아직 구현되지 않음:

- 소개 페이지
- 비 애니메이션
- 빗소리 토글
- 페이지 이동 중 지속되는 오디오
- RSS
- 검색
